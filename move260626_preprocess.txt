[진단] (인스펙터 실측)
------------------------------------------------------------------------
- fps 혼재      : 30fps x144 + 32~60fps 산발 (26종). 30±10% 이탈 103개
- pose jump     : 22  (SLAM teleport, 값 손상)
- rgb non-mono  : 13  (grid 시간역순)
- depth non-mono: 68  / gripper non-mono: 18  (경미)
- gaps          : rgb110 depth140 pose103 gripper85 (대부분 2x, 경미)
- 차원          : rgb 960x720x3 / depth 256x192 / pose 6D / gripper 1D  -> 전부 일관 OK

------------------------------------------------------------------------
[결정 / 처리]
------------------------------------------------------------------------
1) 제외 (손상, 복구 불가) = pose_jump ∪ rgb_nonmono = 34개
      - pose jump : 포즈 값 teleport -> 못 살림
      - rgb non-mono : grid 깨짐 -> delta 오염 (정렬로 일부 복구 가능하나 안전하게 제외)

2) fps -> 전체 30fps 로 리샘플 (제외 아님! 멀쩡한 데이터 회수)
      - 30fps 균일 grid 재격자화:
          RGB / depth (이미지) -> nearest 프레임
          pose / gripper (연속) -> interp (pos linear, rot slerp)
      - 전부 >=30fps 이므로 순수 다운샘플 (업샘플 없음)
      - 주의: 라벨만 30 으로 바꾸는 것 아님. 실제 재격자화 필요 (align 에 리샘플 단계 추가)

3) depth / gripper non-mono -> timestamp 오름차순 정렬 + 중복제거로 흡수 (제외 X)

4) gaps -> 관용 (필요 시 gap 지점 보간 채움)

5) 차원 -> 무처리 (일관)

------------------------------------------------------------------------
[이미지 방향 보정]  (변환 시 bake -- augmentation 아닌 결정적 보정)
------------------------------------------------------------------------
- 현상 : raw RGB 가 시계방향(CW) 90도 돌아가 있음  -> 반시계(CCW) 90도 회전 필요
- 어디서 : 변환(umi2lerobot) 시 bake
    * augmentation(랜덤)이 아니라 결정적 "방향 보정"이라 원본저장 예외 -> 데이터셋을 upright 로 저장
    * LIBERO 선례도 dataset 에 회전을 bake + inference 에서 맞춤 (env_processor.mdx)
- depth : 같은 카메라라 동일 CCW 90도 회전 (RGB 와 픽셀 정합 유지)
- shape 변경 주의 (feature schema 에 반영):
    RGB   960x720x3 -> 720x960x3
    depth 256x192   -> 192x256
- ★ inference 일관성 : 실기 카메라도 같은 CW 90도로 들어오므로,
  추론 관측 파이프라인에서 동일 CCW 90도를 적용해야 함 (train == inference 계약)
